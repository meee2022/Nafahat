# 📘 الدليل الكامل: نشر تطبيق Expo على App Store (خطوة بخطوة بالأوامر)

> مرجع عملي بكل الأوامر اللي استخدمناها + كل مشكلة قابلتنا وحلّها.
> صالح لأي مشروع **Expo / React Native** تاني.

---

## 🧱 0) المتطلبات قبل ما تبدأ

| المتطلب | ملاحظة |
|---|---|
| **حساب Apple Developer** | مدفوع $99/سنة — إجباري للنشر |
| **حساب Expo (EAS)** | مجاني — من expo.dev |
| **Mac + Xcode** | كامل من App Store (للبناء المحلي) |
| **CocoaPods** | بيجي مع Xcode غالباً، أو `brew install cocoapods` |
| **Node** | يُفضّل LTS (20 أو 22). ملاحظة: v26 جديد وممكن يعمل مشاكل |

تأكّد إن الأدوات موجودة:
```bash
xcodebuild -version          # Xcode موجود
pod --version                # CocoaPods
node -v                      # Node
npx eas-cli --version        # EAS CLI (بنستخدمه عبر npx)
```

## ⚙️ 1) إعداد المشروع لأول مرة

### أ) لو المشروع جه من حد تاني (مضغوط/واتساب) — مشاكل الصلاحيات
**مشكلة:** أخطاء `EACCES` أو `operation not permitted` عند البناء.

**السبب:** الملفات فقدت صلاحية التشغيل + ماك حاطط عليها "حجر" (quarantine).

**الحل:**
```bash
cd "مسار/المشروع"

# 1) رجّع صلاحية التشغيل لملفات node_modules
chmod +x node_modules/expo/bin/cli
chmod +x node_modules/.bin/*
find node_modules -type f -path '*/bin/*' -exec chmod +x {} +

# 2) شيل خاصية "الحجر" من المشروع كله
xattr -dr com.apple.quarantine .
```

### ب) ربط المشروع بحساب EAS بتاعك
**مشكلة:** `Entity not authorized` — المشروع متربوط بحساب EAS تاني.

**الحل:** اعمل مشروع EAS جديد تحت حسابك:
```bash
npx eas-cli login                     # سجّل دخول حسابك

# لو فيه projectId قديم في app.json، شيله من:
#   "extra": { "eas": { "projectId": "..." } }   → خليها  "eas": {}

npx eas-cli init --force              # يعمل مشروع جديد ويحط projectId جديد
```

## 🔐 4) إضافة تسجيل الدخول Apple / Google (اختياري)

### Apple Sign-in:
```bash
npx expo install expo-apple-authentication
```
في `app.json`:
```json
"ios": { "usesAppleSignIn": true },
"plugins": [ "expo-apple-authentication", ... ]
```

### Google Sign-in:
```bash
npx expo install @react-native-google-signin/google-signin
```
- اعمل **iOS Client ID** + **Web Client ID** من [Google Cloud Console](https://console.cloud.google.com) ▸ APIs & Services ▸ Credentials.
- في `app.json` plugins:
```json
["@react-native-google-signin/google-signin", { "iosUrlScheme": "com.googleusercontent.apps.<عكس-iOS-client-id>" }]
```

> ⚠️ أبل بيشترط: لو حطيت Google sign-in، **لازم** تحط Apple sign-in كمان (قاعدة 4.8).
> ⚠️ أي تطبيق فيه إنشاء حساب **لازم** يكون فيه **حذف حساب** من جوه التطبيق (قاعدة 5.1.1).

## 🔢 5) أرقام النسخة (مهم جداً للتحديثات)

لو التطبيق **منشور قبل كده**، لازم النسخة الجديدة تكون **أعلى**.

اعرف نسخة الستور من: App Store Connect ▸ التطبيق ▸ Version Information (مثلاً `1.0 (3)`).

في `app.json`:
```json
"version": "1.0.1",                 // أعلى من الماركتنج فيرجن المنشور
"ios": { "buildNumber": "5" }       // أعلى من آخر build رقم
```
في `eas.json` (للتحكم اليدوي في الأرقام):
```json
"cli": { "appVersionSource": "local" }
```
> كل بناء جديد ترفعه = لازم تزوّد `buildNumber` (مينفعش رقم مكرّر).

## 🌐 6) متغيّرات البيئة في نسخة production (مشكلة شائعة!)

**مشكلة:** التطبيق يشتغل في التطوير بس في نسخة الستور يقول "تعذّر الاتصال" — لأن نسخ production **مبتقراش `.env`**.

**الحل:** حُط متغيّرات `EXPO_PUBLIC_*` صريحة في `eas.json` لكل profile:
```json
"build": {
  "production": {
    "env": {
      "EXPO_PUBLIC_CONVEX_URL": "https://xxx.convex.cloud",
      "EXPO_PUBLIC_SENTRY_DSN": "https://..."
    },
    "ios": {}
  }
}
```
**للتأكد إن المتغيّر اتضمّن في الـ ipa:**
```bash
mkdir -p /tmp/chk && cd /tmp/chk
unzip -o -q "مسار/build-XXXX.ipa" "Payload/*/main.jsbundle"
grep -c "xxx.convex.cloud" Payload/*/main.jsbundle   # لازم يطلع رقم > 0
```

## 🖼️ 7) شاشة البداية (Splash)

في `app.json` plugin `expo-splash-screen`:
```json
["expo-splash-screen", {
  "image": "./assets/splash.png",      // اللوجو بس (مش تصميم ملء الشاشة)
  "imageWidth": 390,                   // عرض اللوجو — كبّره لو طلع صغير
  "resizeMode": "contain",
  "backgroundColor": "#021e12"         // نفس لون خلفية اللوجو عشان يندمج
}]
```
> **مهم:** الصورة تكون **اللوجو بس** متمركز، مش تصميم ملء الشاشة. والخلفية = لون موحّد.
> لو اللوجو ظهر صغير → زوّد `imageWidth`.

## 🏗️ 8) البناء المحلي (eas build --local)

### نسخة تطوير (للتجربة على جهازك + تعديل فوري):
```bash
EAS_NO_VCS=1 npx eas-cli build --platform ios --profile development --local
```

### نسخة الستور (production):
```bash
EAS_NO_VCS=1 FASTLANE_XCODEBUILD_SETTINGS_TIMEOUT=120 FASTLANE_XCODEBUILD_SETTINGS_RETRIES=5 \
  npx eas-cli build --platform ios --profile production --local
```

**شرح المتغيّرات:**
| المتغيّر | الفايدة |
|---|---|
| `EAS_NO_VCS=1` | يبني من الملفات الحالية من غير ما تحتاج git commit |
| `FASTLANE_XCODEBUILD_SETTINGS_TIMEOUT=120` | يحل مشكلة `xcodebuild -showBuildSettings timed out` |
| `FASTLANE_XCODEBUILD_SETTINGS_RETRIES=5` | محاولات أكتر |

**أول مرة هيسألك:** تسجيل دخول Apple + يعمل Distribution Certificate + Provisioning Profile + APNs key (لو فيه إشعارات) → جاوب **Yes** على كله، وهو بيعملهم ويحفظهم.

## 📲 9) التركيب على الآيفون (للنسخة development/preview بس)

```bash
# اعرف UDID الجهاز الموصّل:
xcrun xctrace list devices | grep iPhone

# سجّل الجهاز عند Apple (مرة واحدة، للنسخ الـ ad-hoc):
npx eas-cli device:create

# ركّب الـ ipa مباشرة (من غير Xcode):
xcrun devicectl device install app --device <UDID> "build-XXXX.ipa"

# لو غيّرت الـ splash: امسح القديم الأول (عشان كاش iOS) ثم ركّب:
xcrun devicectl device uninstall app --device <UDID> com.nafahat.app
xcrun devicectl device install app --device <UDID> "build-XXXX.ipa"
```
> نسخة **production مبتتركّبش بالكابل** — بس عن طريق TestFlight (خطوة 11).

### وضع التطوير الفوري (Fast Refresh):
```bash
npx expo start --dev-client
```
بعدها أي تعديل في الكود (مش native) يظهر فوراً من غير بناء.

## ☁️ 10) أنواع البناء (الفرق)

| النوع | الغرض | يتركّب بالكابل؟ |
|---|---|---|
| `development` | تطوير + تعديل فوري | ✅ |
| `preview` | نسخة شبه الستور للتجربة | ✅ (ad-hoc) |
| `production` | الرفع للستور / TestFlight | ❌ (TestFlight بس) |

**القاعدة:** native (splash، أصوات، مكتبات، أيقونة، أذونات) = محتاج **بناء**.
كود (شاشات، نصوص، منطق) = **Fast Refresh فوري** بدون بناء.

## 🚀 11) الرفع لـ TestFlight / App Store

```bash
npx eas-cli submit --platform ios --path "build-XXXX.ipa"
```
- أول مرة: **Generate a new App Store Connect API Key?** → **Yes** → اختار **ADMIN**.
- بيرفع الملف لـ App Store Connect.

بعد الرفع (على [appstoreconnect.apple.com](https://appstoreconnect.apple.com)):
- التطبيق ▸ **TestFlight** → استنى المعالجة (~١٠-٣٠ دقيقة)
- **Internal Testing** → ضيف المختبرين (بدون مراجعة من أبل، فوري)
- المختبرين ينزّلوا تطبيق **TestFlight** ويجرّبوا

للنشر النهائي: التطبيق ▸ نسخة جديدة → اختار الـ build → املأ البيانات → **Submit for Review**.

## 🧯 12) جدول المشاكل والحلول (مرجع سريع)

| المشكلة | الحل |
|---|---|
| `zsh: command not found: eas` | استخدم `npx eas-cli ...` |
| `EACCES` على node_modules | `chmod +x node_modules/expo/bin/cli` + `.bin/*` |
| `operation not permitted` | `xattr -dr com.apple.quarantine .` |
| `Entity not authorized` (EAS) | شيل projectId القديم + `npx eas-cli init --force` |
| `spawn fastlane ENOENT` | `brew install fastlane` |
| `xcodebuild -showBuildSettings timed out` | `FASTLANE_XCODEBUILD_SETTINGS_TIMEOUT=120` |
| `pod install ... React-Core-prebuilt` (expo run:ios) | استخدم `eas build --local` بدل `expo run:ios` |
| splash لوجو صغير | زوّد `imageWidth` + خلّي الصورة اللوجو بس |
| splash مش بيتحدّث | امسح التطبيق وركّبه نضيف (كاش iOS) |
| "تعذّر الاتصال" في production | حُط `EXPO_PUBLIC_*` في `eas.json` env |
| أبل يرفض: نسخة مكرّرة | زوّد `version` و `buildNumber` |
